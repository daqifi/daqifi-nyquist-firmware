  - Evidence: SD_MOUNT_MAX_RETRIES = 3:26, mount failure loop in MOUNT_DISK:493, SD task cadence is 1 ms in app_freertos.c:290.
  - Impact: if mount needs more than a few ms after insert/power-up, mode is forced to NONE (sd_card_manager.c:498) and logging won’t resume until re-armed.
  - Suggestion: switch from count-only retries to time-budget + backoff (example: 500-1000 ms total, 10-20 ms delay between attempts).

  2. Medium: unconditional completion semaphore signal in ERROR can leave stale token

  - Evidence: ERROR always gives opCompleteSemaphore:1149, wait path consumes one token in WaitForCompletion:1229, but no explicit drain before starting a new async
    operation in UpdateSettings:1212.
  - Impact: future LIST/DELETE/FORMAT waits can return immediately on an old signal.
  - Suggestion: drain semaphore before launching a new waited operation, or only signal completion in ERROR for operations that actually use WaitForCompletion.
