# Refactor BQ24297_Read_I2C: Replace 0xFF Sentinel with Bool Return

## Context

`BQ24297_Read_I2C(uint8_t reg)` returns `uint8_t` with `0xFF` as the error sentinel. But `0xFF` is valid register data (e.g. REG00 with HIZ+max VINDPM+3A IINLIM). The underlying I2C driver (`DRV_I2C_WriteReadTransfer`) already returns `bool` for success/failure — we should propagate that instead of collapsing it into a data value. `BQ24297_Write_I2C` already returns `bool` correctly.

Additionally, 6 call sites have **no error check at all** before using the read value for read-modify-write — these are existing bugs that this refactor fixes.

## API Change

```c
// Old:
uint8_t BQ24297_Read_I2C(uint8_t reg);           // 0xFF = error (ambiguous)
// New:
bool BQ24297_Read_I2C(uint8_t reg, uint8_t *out); // false = error, *out unchanged
```
