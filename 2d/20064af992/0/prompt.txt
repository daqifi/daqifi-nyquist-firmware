# Robust REG09 Fault Handling for BQ24297

## Context

BQ24297 REG09 is a latched fault register — reading it clears all latched faults. The current firmware has three problems:

1. **Latched faults silently lost**: Every `BQ24297_UpdateStatus()` call does a double-read (clear latched, read current). The latched faults from the first read are discarded. If a transient fault occurred between reads, it vanishes.

2. **SCPI diagnostic commands inconsistent**: `SCPI_GetBQRegisters` reads REG09 once in a loop (clearing latches), then re-reads much later. `SCPI_GetBQDiagnostics` does a single read after `UpdateStatus` already cleared latches.

3. **No runtime NTC monitoring after boot**: `BQ24297_UpdateBatteryStatus()` is called once at init, never again. If a thermistor fault develops at runtime, charging is never disabled. (Future: issue #193 will add INT-pin-driven fault monitoring to address this without I2C polling.)

### Datasheet Reference (BQ24297 REG09, Table 9-16, p41)

```
Bit 7: WATCHDOG_FAULT    Bit 6: OTG_FAULT
Bit 5: CHRG_FAULT[1]     Bit 4: CHRG_FAULT[0]
Bit 3: BAT_FAULT          Bit 2: Reserved (always 0)
Bit 1: NTC_FAULT[1]      Bit 0: NTC_FAULT[0]
```

- NTC is 2-bit field [1:0]: 00=Normal, 01=Hot, 10=Cold, 11=Hot+Cold
- **Footnote (2)**: "All register bits in REG09 are latched fault. First time read clears the previous fault and second read updates fault register to any fault that still presents."
- Current firmware 2-bit mask `0x03` is CORRECT per datasheet. The SCPI 3-bit mask `0x07` harmlessly includes reserved bit 2.

### User Requirements
- Never silently clear faults — accumulate them in a global
- System should act on accumulated faults as needed
- At boot, check NTC: COLD = no battery (disable charging), HOT = disable charging for safety
- Runtime NTC re-checking deferred to issue #193 (INT-pin-driven, no I2C polling)

```c
bool watchdog_faultAccum;
bool otg_faultAccum;
enum eChargeFault chgFaultAccum;
bool bat_faultAccum;
enum eNTCFault ntcFaultAccum;
```

Add a function to clear them:
```c
void BQ24297_ClearAccumulatedFaults(void);
```

### 2. Centralize all REG09 reads through a single function

**File: `firmware/src/HAL/BQ24297/BQ24297.c`**

Create a new internal helper that EVERY REG09 read goes through:

```c
/**
 * Read REG09 and accumulate faults.
 * First read captures+clears latched faults, second read gets current.
 * Both reads feed into accumulated fault tracking.
 * @param[out] latched  Raw latched REG09 value (NULL to skip)
 * @param[out] current  Raw current REG09 value (NULL to skip)
 * @return true if at least one read succeeded
 */
static bool BQ24297_ReadFaultReg(uint8_t *latched, uint8_t *current);
```

Implementation:
- Read REG09 → store as `latchedVal`, OR into accumulated faults
- Read REG09 again → store as `currentVal`, OR into accumulated faults
- Decode `currentVal` into `pData->status` fields (existing behavior)
- If first read fails → return false, don't touch fields
- If second read fails → use latchedVal for status fields, log error

Accumulation logic (applied to BOTH latched and current reads):
```c
static void AccumulateFaults(uint8_t reg09) {
    if (reg09 & 0x80) pData->status.watchdog_faultAccum = true;
    if (reg09 & 0x40) pData->status.otg_faultAccum = true;
    uint8_t chgF = (reg09 >> 4) & 0x03;
    if (chgF > pData->status.chgFaultAccum) pData->status.chgFaultAccum = chgF;
    if (reg09 & 0x08) pData->status.bat_faultAccum = true;
    uint8_t ntcF = reg09 & 0x03;
    if (ntcF != NTC_FAULT_NORMAL) pData->status.ntcFaultAccum = ntcF;
}
```

### 3. Update `BQ24297_UpdateStatus()` to use the centralized reader

Replace current REG09 double-read block (lines 238-252) with call to `BQ24297_ReadFaultReg(NULL, NULL)`.

### 4. Update SCPI commands to use centralized reader

**File: `firmware/src/services/SCPI/SCPIInterface.c`**

**SCPI_GetBQRegisters**: Instead of reading REG09 in the loop (which clears latches without accumulating), skip index 9 in the loop and call `BQ24297_ReadFaultReg(&latched, &current)` separately to get both values properly accumulated.

**SCPI_GetBQDiagnostics**: Replace the single `BQ24297_Read_I2C(0x09)` with `BQ24297_ReadFaultReg(&latched, &current)`. Display current faults (since `UpdateStatus` was just called, latched should be clear unless a new fault appeared).

### 5. Expose accumulated faults in SCPI diagnostics

**File: `firmware/src/services/SCPI/SCPIInterface.c`** — `SCPI_GetBQDiagnostics`

Add a section after the current REG09 output showing accumulated faults:

```
[Accumulated Faults]
  Watchdog=0 | OTG=0 | CHG=Normal (0) | BAT=0 | NTC=Ok (0)
```

### 6. Clear accumulated faults at appropriate times

- On `BQ24297_Config_Settings()` (boot): clear all accumulated fields after init
- Expose via SCPI: Add `SYST:POW:BQ:FAULTCLR` command to manually clear
